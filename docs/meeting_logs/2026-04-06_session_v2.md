## 1. 논의 요약

v1 세션에서 Gemini 이중처리 문제를 수정했으나, 이후 테스트에서 연쇄적으로 추가 문제가 발견됨.

**Gmail 인증 문제 (가장 큰 이슈):**
- "서버에 연결할 수 없습니다" 에러가 반복 → 여러 원인이 중첩
- OAuth 토큰 만료 (Google 테스트 모드 7일 제한) → 재연동 시도
- 재연동해도 설정에서 "미연동" 표시 → `ensureUser()`가 DB `id`를 `clerkId`로 검색하여 유령 사용자 생성, 토큰이 잘못된 사용자에 저장
- 재연동 시 옛 만료 refresh_token이 그대로 유지 → 새 토큰이 저장 안 됨
- Railway cold start 시 502 반환에 CORS 헤더 없음 → 브라우저가 CORS 차단
- Service Worker가 옛 코드를 캐시하여 프론트엔드 수정이 반영 안 됨

**이메일/일정 포맷:**
- 이메일 브리핑이 여전히 줄글 → Sonnet 프롬프트의 출력 포맷 지시 강화
- 일정 응답이 단순 → 이메일 맥락 연계 + 가독성 개선

**회의/태스크 개선:**
- 모바일 PWA 화면 잘림, 태스크 연속 입력, 토스트 위치 등 다수 UX 수정
